Introduccion

JavaScript es un lenguaje de secuencias de comandos orientado a objetos y multiplataforma. Eso es un lenguaje pequeño y ligero. Dentro de un entorno anfitrión (para ejemplo, un navegador web), JavaScript se puede conectar a los objetos de su entorno para proporcionar control programático sobre ellos.

JavaScript contiene una biblioteca estándar de objetos, como Array, Fecha y Matemáticas, y un conjunto básico de elementos del lenguaje como operadores, estructuras de control y declaraciones. Core JavaScript puede ser extendido para una variedad de propósitos completándolo con objetos adicionales; por ejemplo:

  • JavaScript del lado del cliente amplía el lenguaje central al proporcionar objetos para controlar un navegador y su modelo de objetos de documento (DOM). Por ejemplo, las extensiones del lado del cliente permiten que una aplicación coloque elementos en un formulario HTML y responder a eventos de usuario como el mouse clics, entrada de formulario y navegación de página.
  • JavaScript del lado del servidor extiende el lenguaje central al proporcionar objetos relevantes para ejecutar JavaScript en un servidor. Por ejemplo, Las extensiones del lado del servidor permiten que una aplicación se comunique con un base de datos, proporcionan continuidad de la información de una invocación a otra de la aplicación, o realizar manipulaciones de archivos en un servidor.

JavaScript & Java

JavaScript y Java son similares en algunos aspectos, pero fundamentalmente diferente en algunos otros. El lenguaje JavaScript se parece a Java pero no tiene la escritura estática de Java y la verificación de tipo fuerte. JavaScript sigue la mayoría de la sintaxis de expresiones de Java, convenciones de nomenclatura y constructos básicos de flujo de control que fue la razón por la que fue renombrado de LiveScript a JavaScript.

En contraste con el sistema de clases en tiempo de compilación de Java construido por declaraciones, JavaScript admite un sistema de tiempo de ejecución basado en un pequeño número de tipos de datos que representan numérico, booleano y de cadena valores. JavaScript tiene un modelo de objetos basado en prototipos en lugar del modelo de objetos basado en clases más común. El modelo basado en prototipos proporciona herencia dinámica; es decir, lo que se hereda puede variar para objetos individuales. JavaScript también admite funciones sin cualquier requisito declarativo especial. Las funciones pueden ser propiedades de objetos, que se ejecutan como métodos vagamente tipados.

JavaScript es un lenguaje de forma muy libre en comparación con Java. Tu no tiene que declarar todas las variables, clases y métodos. Usted no tiene preocuparse por si los métodos son públicos, privados o protegido, y no tiene que implementar interfaces. Variables los parámetros y los tipos de retorno de función no se escriben explícitamente.

Variables

Utiliza variables como nombres simbólicos para valores en su aplicación. Los nombres de las variables, llamados identificadores, se ajustan a ciertas reglas.

Un identificador de JavaScript debe comenzar con una letra, un guión bajo (_) o un signo de dólar ($); los caracteres siguientes también pueden ser dígitos (0-9). Debido a que JavaScript distingue entre mayúsculas y minúsculas, las letras incluyen los caracteres de la "A" a la "Z" (mayúsculas) y los caracteres de la "a" a la "z" (minúsculas).

Puede utilizar letras ISO 8859-1 o Unicode como å y ü en identificadores. También puede utilizar las secuencias de escape Unicode como caracteres en identificadores. Algunos ejemplos de nombres legales son Number_hits, temp99 y _name.

Declaración de variables
Puede declarar una variable de tres formas:

Con la palabra clave var. Por ejemplo, var x = 42. Esta La sintaxis se puede utilizar para declarar variables tanto locales como globales.

Simplemente asignándole un valor. Por ejemplo, x = 42. Esto siempre declara una variable global. Eso genera una advertencia estricta de JavaScript. No deberías usar esto variante.

Con la palabra clave dejar. Por ejemplo, let y = 13. Esta La sintaxis se puede utilizar para declarar una variable local de alcance de bloque. Ver Alcance variable a continuación.

Alcance variable

Cuando declaras una variable fuera de cualquier función, se llama variable global, porque está disponible para cualquier otro código en el documento actual. Cuando declaras una variable dentro de una función, se llama una variable local, porque está disponible sólo dentro de ese función.

JavaScript antes de ECMAScript 2015 no tiene declaración de bloque alcance; más bien, una variable declarada dentro de un bloque es local al función (o alcance global) en la que reside el bloque. Para ejemplo, el siguiente código registrará 5, porque el alcance de x es el función (o contexto global) dentro de la cual se declara x, no el block, que en este caso es una sentencia if.

if (true) { var x = 5; } console.log(x); // 5

Este comportamiento cambia cuando se usa la declaración let introducida en ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Variables globales

Las variables globales son de hecho propiedades del objeto global. En la web páginas el objeto global es la ventana, por lo que puede configurar y acceder global variables usando la sintaxis window.variable.

En consecuencia, puede acceder a las variables globales declaradas en una ventana. o marco de otra ventana o marco especificando la ventana o nombre del marco. Por ejemplo, si una variable llamada phoneNumber es declarada en un documento, puede hacer referencia a esta variable desde un iframe como parent.phoneNumber.

Constantes

Puede crear una constante de solo lectura con nombre con la palabra clave const. La sintaxis de un identificador constante es la misma que la de una variable identificador: debe comenzar con una letra, un guión bajo o un signo de dólar y puede contener caracteres alfabéticos, numéricos o de subrayado.

const PI = 3.14;

Una constante no puede cambiar el valor a través de la asignación o volver a declararse mientras se ejecuta el script. Tiene que inicializarse a un valor.

Las reglas de alcance para las constantes son las mismas que las del bloque let variables de alcance. Si se omite la palabra clave const, el identificador es se supone que representa una variable.

declaración if...else
Utilice la instrucción if para ejecutar una instrucción si una condición lógica es verdadero. Utilice la cláusula else opcional para ejecutar una declaración si el la condición es falsa. Una declaración if tiene el siguiente aspecto: if (condition) { statement_1; } else { statement_2; } La condición puede ser cualquier expresión que se evalúe como verdadera o falsa. Ver Booleano para una explicación de lo que se evalúa como verdadero y falso. Si la condición se evalúa como verdadera, se ejecuta la instrucción_1; de lo contrario, declaración_2 se ejecuta. declaración_1 y declaración_2 pueden ser cualquier declaración, incluidas más declaraciones if anidadas.

También puede componer las declaraciones usando else if to have multiple condiciones probadas en secuencia, como sigue:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } En el caso de múltiples condiciones, solo la primera condición lógica que se evalúa como verdadero se ejecutará. Para ejecutar múltiples declaraciones, agrúpelas dentro de una declaración de bloque ({ ... }) . En En general, es una buena práctica utilizar siempre sentencias de bloque, especialmente al anidar declaraciones if: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } Es aconsejable no utilizar asignaciones simples en un condicional expresión, porque la asignación se puede confundir con la igualdad cuando echando un vistazo al código. Por ejemplo, no utilice el siguiente código: if (x = y) { /* statements here */ } Si necesita usar una asignación en una expresión condicional, una práctica común es poner paréntesis adicionales alrededor de la tarea. Por ejemplo: if ((x = y)) { /* declaraciones aquí */ }
declaración while
Una declaración while ejecuta sus declaraciones siempre que se especifique la condición se evalúa como verdadera. Una declaración while tiene el siguiente aspecto: Una declaración while (condition) Si la condición se vuelve falso, la declaración dentro del bucle deja de ejecutarse y el control pasa a la declaración que sigue al bucle.

La prueba de condición ocurre antes de que se ejecute la instrucción en el ciclo. Si la condición devuelve verdadera, la declaración se ejecuta y la la condición se prueba de nuevo. Si la condición devuelve falso, ejecución detiene y el control se pasa a la instrucción que sigue a while.

Para ejecutar varias declaraciones, use una declaración de bloque ({ ... }) a agrupe esas declaraciones.

Ejemplo:

El siguiente ciclo while se repite siempre que n sea menor que tres:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Con cada iteración, el ciclo incrementa n y agrega ese valor ax. Por tanto, xyn toman los siguientes valores:

  • Después del primer pase: n = 1 and x = 1
  • Después del segundo pase: n = 2 and x = 3
  • Después del tercer pase: n = 3 and x = 6

Después de completar el tercer pase, la condición n < 3 ya no es verdadero, entonces el ciclo termina.

Declaraciones de funciones
Una definición de función (también llamada declaración de función, o función declaración) consta de la palabra clave de función, seguida de:
  • El nombre de la función.
  • Una lista de argumentos de la función, entre paréntesis y separado por comas.
  • Las declaraciones de JavaScript que definen la función, incluidas en corchetes, {}.

Por ejemplo, el siguiente código define una función simple llamada cuadrado:

Funcion cuadrado(numero) { return numero * numero; }

La función cuadrado toma un argumento, llamado número. La función Consiste en una declaración que dice devolver el argumento de la función (es decir, número) multiplicada por sí misma. El regreso declaración especifica el valor devuelto por la función.

return numero * numero;

Los parámetros primitivos (como un número) se pasan a funciones mediante valor; el valor se pasa a la función, pero si la función cambia el valor del parámetro, este cambio no se refleja globalmente o en la función de llamada.
